### Get KYC profile

```http
GET /kyc/profile
```

Retrieves the KYC profile for the authenticated user, including all associated personal info, address, documents, and employment data.


**Response**

| Field | Type | Description |
| ----- | ---- | ----------- |
| `kycProfileId` | `string` | KYC profile ID |
| `bkeyUserId` | `string` | User ID |
| `status` | `not_started \| in_progress \| pending_review \| completed \| rejected` | KYC profile status |
| `completedAt` | `string \| null` | Completion timestamp |
| `personalInfo` | `object` | Personal information |
| `address` | `object` | Address information |
| `identificationNumbers` | `object[]` | Identification numbers |
| `identificationDocuments` | `object[]` | Identification documents |
| `proofOfAddressDocuments` | `object[]` | Proof of address documents |
| `biometricDocuments` | `object[]` | Biometric documents |
| `createdAt` | `string` | Creation timestamp |
| `updatedAt` | `string` | Last update timestamp |

### Update KYC profile with partial data

```http
PATCH /kyc/profile
```

Accepts partial KYC data and merges with existing profile. All fields are optional. Returns what was saved and if profile can be activated.


**Request body**

| Field | Type | Required | Description |
| ----- | ---- | -------- | ----------- |
| `personalInfo` | `object` | No |  |
| `address` | `object` | No |  |
| `employment` | `object` | No |  |
| `sourceOfFunds` | `salary \| business \| investments \| pension \| government \| inheritance \| savings` | No |  |
| `estimatedMonthlyVolume` | `number` | No |  |
| `accountPurpose` | `personal \| business \| investment` | No |  |
| `actingAsIntermediary` | `boolean` | No |  |
| `identificationNumbers` | `object[]` | No |  |
| `bridgeSignedAgreementId` | `string` | No |  |


**Response**

| Field | Type | Description |
| ----- | ---- | ----------- |
| `success` | `boolean` |  |
| `saved` | `object` |  |
| `errors` | `object[]` |  |
| `canActivate` | `boolean` |  |
| `missing` | `string[]` |  |


**Response**

| Field | Type | Description |
| ----- | ---- | ----------- |

### Upload physical identification document with images

```http
POST /kyc/identification-documents
```

Upload a physical identification document using multipart/form-data. Send files as "files" field and document data as form fields. Front image is required, back image is optional.


**Request body**

| Field | Type | Required | Description |
| ----- | ---- | -------- | ----------- |
| `type` | `passport \| drivers_license \| national_id \| government_id \| other` | Yes | Type of identification document |
| `documentNumber` | `string` | Yes | Document number |
| `issuingCountry` | `string` | Yes |  |
| `expirationDate` | `string` | No | Document expiration date |
| `issueDate` | `string` | No | Document issue date |


**Response**

| Field | Type | Description |
| ----- | ---- | ----------- |
| `id` | `string` | Document unique identifier |
| `type` | `passport \| drivers_license \| national_id \| government_id \| other` | Type of identification document |
| `documentNumber` | `string` | Document number |
| `issuingCountryCode` | `string` | Country that issued the document |
| `expirationDate` | `string` | Document expiration date |
| `issueDate` | `string` | Document issue date |
| `status` | `pending \| verified \| rejected \| expired` | Document verification status |
| `verificationNotes` | `string` | Notes from verification process |
| `verifiedAt` | `string` | Verification timestamp |
| `verifiedBy` | `string` | Verifier identifier |
| `createdAt` | `string` | Creation timestamp |
| `updatedAt` | `string` | Last update timestamp |

### Lookup BVN data without saving


| Field | Type | Description |
