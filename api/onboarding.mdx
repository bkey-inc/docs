## Onboard

### Start Nigeria onboarding flow

```http
POST /onboarding/start-nigeria
```

Starts the Nigeria-specific onboarding workflow with BVN verification, NGN wallet, and Anchor + Bridge integration.


**Request body**

| Field | Type | Required | Description |
| ----- | ---- | -------- | ----------- |
| `bvn` | `string` | Yes |  |
| `usdWalletAddress` | `string` | Yes |  |
| `usdWalletIndex` | `number` | Yes | USD wallet index |
| `ngnWalletAddress` | `string` | Yes |  |
| `ngnWalletIndex` | `number` | Yes | NGN wallet index |


**Response**

| Field | Type | Description |
| ----- | ---- | ----------- |
| `workflowId` | `string` | Workflow ID for the onboarding process |
| `isNigeria` | `boolean` | Indicates this is Nigeria onboarding |
| `status` | `object` | Onboarding status indicators |

### Get onboarding status

```http
GET /onboarding/status
```

Get the status of the onboarding process


**Response**

| Field | Type | Description |
| ----- | ---- | ----------- |
| `anchorStatus` | `not_started \| pending \| active \| resubmission_required \| rejected` | Anchor onboarding status |
| `bridgeStatus` | `not_started \| pending \| active \| resubmission_required \| rejected` | Bridge onboarding status |
| `moneriumStatus` | `not_started \| pending \| active \| resubmission_required \| rejected` | Monerium onboarding status |
| `bridgeRetryAllowed` | `boolean` | Whether the user can retry Bridge KYC (only present if rejected) |
| `bridgeRejectionReasons` | `string[]` | Bridge rejection reasons if status is resubmission_required |
| `bridgeRejectionCues` | `string[]` | User-facing guidance cues for Bridge KYC rejection |
| `paytrieStatus` | `not_started \| pending \| active \| resubmission_required \| rejected` | PayTrie onboarding status (Canada) |
| `paytrieRejectionReason` | `string` | PayTrie rejection reason if status is resubmission_required |

## Tier

### Get all available tiers

```http
GET /tier
```

Returns all available tier levels (Bronze, Silver, Gold, Platinum) with their configurations and pricing. Requires authentication.


**Response**

| Field | Type | Description |
| ----- | ---- | ----------- |
| `tiers` | `object[]` |  |


---


### Get current user tier

```http
GET /tier/current
```

Returns the tier for the authenticated user. Currently hardcoded to BRONZE tier.


**Response**

| Field | Type | Description |
| ----- | ---- | ----------- |
| `name` | `BRONZE \| SILVER \| GOLD \| PLATINUM` |  |
| `acquisition` | `object` | How to obtain this tier |
| `fees` | `object` | Base fees for bank and crypto operations |
| `features` | `object` | Virtual Bank Account features and their costs |
