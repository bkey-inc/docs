### List Conduit customers

```http
GET /conduit/customers
```

Lists all customers from Conduit. Use this to find the customer ID for counterparty creation.


**Query parameters**

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `pageSize` | `number` | No | Number of items per page (default: 50) |
| `pageAfter` | `string` | No | Cursor for pagination (customer ID to start after) |

### Create customer onboarding draft

```http
POST /conduit/customers/onboarding
```

Creates a new customer onboarding draft with Section 1 (Business Identity) data. Returns an internal onboarding ID for subsequent updates.


**Request body**

| Field | Type | Required | Description |
| ----- | ---- | -------- | ----------- |
| `businessLegalName` | `string` | Yes | Legal name of the business |
| `country` | `string` | Yes | ISO 3166-1 alpha-3 country code |
| `registeredDate` | `string` | Yes | Business registration date (YYYY-MM-DD) |
| `website` | `string` | No | Company website URL |
| `taxIdentificationNumber` | `string` | Yes | Tax identification number (EIN) |
| `businessEntityId` | `string` | Yes | Business entity ID |
| `email` | `string` | Yes | Business email address |
| `phone` | `string` | Yes | Phone number with country code |


**Response**

| Field | Type | Description |
| ----- | ---- | ----------- |
| `onboardingId` | `string` | Internal onboarding record ID |
| `status` | `string` | Current onboarding status |
| `currentStep` | `number` | Current form step |

| Parameter | Type | Required | Description |

| `controlPerson` | `object` | No |  |
| `identityInfo` | `object` | No |  |

### Upload KYB or KYC document

```http
POST /conduit/customers/onboarding/{id}/documents
```

Uploads a document for the onboarding record. Requires purpose ("kyb" for business docs, "kyc" for identity docs), type (e.g. "certificate_of_registration", "passport"), and name. Accepts JPG, PNG, or PDF (max 10MB). Documents are stored locally and uploaded to Conduit during submission.


**Path parameters**

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `id` | `string` | Yes | Internal onboarding ID |

---

### Get sender counterparty

```http
GET /conduit/sender-counterparty
```

Retrieves the sender counterparty for the authenticated user. Returns null if no sender counterparty exists. Used to pre-fill the KYC form.


---


### List receiver counterparties

```http
GET /conduit/receiver-counterparties
```

Lists all added receiver counterparties for the authenticated user. Receiver counterparties represent beneficiaries who receive fiat payments in offramp transactions. Counterparties are scoped to users.


---

### Create receiver counterparty

```http
POST /conduit/counterparties/receiver
```

Creates a receiver counterparty with a bank account as the fiat payment method for the authenticated user. Used for offramp transactions where we pay fiat to the beneficiary. Counterparties are scoped to users.


**Request body**

| Field | Type | Required | Description |
| ----- | ---- | -------- | ----------- |
| `idempotencyKey` | `string` | Yes | Unique key to ensure idempotent counterparty creation |
| `destinationCurrency` | `string` | Yes | Destination currency code |
| `beneficiary` | `object` | Yes |  |
| `senderId` | `string` | No | Conduit counterparty ID of the sender |

### List supported banks

```http
GET /conduit/banks
```

Returns a list of banks supported by Conduit for various payment rails. Can be filtered by country code (ISO 3166-1 alpha-3) or currency code. Currently supports: Brazil (BRL/TED), Colombia (COP/ACH), and China (USD/SWIFT).


**Query parameters**

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `countryCode` | `string` | No | Filter by ISO 3166-1 alpha-3 country code (e.g., BRA, COL, CHN) |
| `currency` | `string` | No | Filter by currency code (e.g., BRL, COP, USD) |

**Response**

| Field | Type | Description |
| ----- | ---- | ----------- |
| `banks` | `object[]` | Supported banks organized by country |
| `total` | `number` | Total number of bank entries |
