## Installation

### 1. Add the dependency

**Monorepo / local path:**

```yaml
# pubspec.yaml
dependencies:
  bkey_uikit:
    path: ../bkey-packages/bkey_uikit
```

**Git (current development branch):**

```yaml
# pubspec.yaml
dependencies:
  bkey_uikit:
    git:
      url: https://github.com/bkey-inc/Bkey-mobile-uikit.git
      ref: extract-components
```

### 2. Run pub get

```bash
flutter pub get
```

### 3. Apply the theme

```dart
import 'package:bkey_uikit/bkey_uikit.dart';
import 'package:flutter/material.dart';

void main() {
  runApp(
    MaterialApp(
      theme: BMoniTheme.darkTheme(),
      home: const MyHomePage(),
    ),
  );
}
```

## Single import

All public API is available from one import:

```dart
import 'package:bkey_uikit/bkey_uikit.dart';
```

## Using it with feature packages

`bkey_uikit` is the design-system layer. Feature packages should depend on it for
generic building blocks, then add their own domain-aware composition on top.

Example:

```yaml
dependencies:
  bkey_uikit:
    path: ../bkey-packages/bkey_uikit
  bmoni_embedded_wallets_cards:
    path: ../bkey-packages/bmoni_embedded_wallets_cards
```

In this setup:

- `bkey_uikit` owns tokens and generic widgets
- `bmoni_embedded_wallets_cards` owns wallet/card-aware widgets and providers
- the host app owns routing, localisation, analytics, and app-specific actions

## Package structure

```
lib/
  bkey_uikit.dart        ← single barrel export
  src/
    tokens/
      colors.dart        ← BMoniColors
      text_styles.dart   ← BMoniTextStyles
      bmoni_theme.dart   ← BMoniTheme
    enums/               ← shared enumerations
    components/          ← reusable widget and layout primitive files
assets/
  fonts/                 ← Rethink Sans TTF files
```
